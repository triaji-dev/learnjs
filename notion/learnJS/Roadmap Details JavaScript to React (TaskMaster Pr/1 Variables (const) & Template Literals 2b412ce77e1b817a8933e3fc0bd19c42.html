A. Konsep Dasar Variables: Fokus pada const

Dalam JavaScript modern (ES6+), kita menggunakan tiga kata kunci untuk mendeklarasikan variabel: var, let, dan const. Dalam pengembangan aplikasi modern, kita sangat dianjurkan menggunakan let dan const.

1. Penggunaan dan Tujuan const (Constant)

Kata kunci const digunakan untuk mendeklarasikan variabel yang merujuk ke suatu nilai yang tidak dapat diubah (reassign) setelah diinisialisasi.

  • Kapan menggunakan const?
    • Untuk data statis yang seharusnya tidak berubah sepanjang aplikasi berjalan (misalnya: judul aplikasi, URL API, batas maksimum).
    • Untuk mendeklarasikan objek dan array yang referensinya tidak akan diganti dengan objek/array lain.

2. Aturan Penting const

Tipe DataPerubahan (Re-assignment)Perubahan (Internal)
Primitif (String, Number, Boolean)TIDAK BISATIDAK BERLAKU
Referensi (Object, Array)TIDAK BISABISA (Properti atau elemen di dalamnya dapat diubah)

Contoh Penggunaan const:

JavaScript

// ðŸ’¡ Data Primitif: Tidak bisa di-reassign
const APP_TITLE = "TaskMaster Pro";
// APP_TITLE = "New Title"; // -> Akan menyebabkan Error!

// ðŸ’¡ Data Struktur (Objek User)
const USER_DATA = {
    id: 'u-456',
    firstName: 'Budi',
    totalTasks: 10
};

// USER_DATA tidak bisa di-reassign (referensi objek tidak bisa diganti):
// USER_DATA = {}; // -> Akan menyebabkan Error!

// Namun, properti di dalam objek BISA diubah:
USER_DATA.totalTasks = 12; // -> Ini diperbolehkan!

B. Template Literals (String Interpolation)

Template Literals adalah fitur ES6 yang memungkinkan kita untuk membuat string dengan cara yang lebih bersih, terutama untuk tujuan:

  1. Multi-line String: Membuat string yang membentang di beberapa baris tanpa perlu karakter \n.
  1. String Interpolation: Menyisipkan nilai variabel (atau ekspresi JavaScript) langsung ke dalam string.

Template Literals dideklarasikan menggunakan tanda backtick (`) dan bukan tanda kutip tunggal (') atau ganda (").

1. Syntax Interpolation

Untuk menyisipkan variabel atau ekspresi, gunakan format: ${variabel_atau_ekspresi}

FiturTanda Kutip BiasaTemplate Literals
Interpolation'Halo, ' + name + '!'`Halo, ${name}!`
Multi-line'Baris 1\nBaris 2'`Baris 1 Baris 2`

2. Membuat Struktur HTML Dasar (Relevansi React: JSX)

Template Literals sangat kuat untuk menyusun blok HTML/UI dinamis dengan data statis. Ini adalah fondasi dasar bagaimana JSX (JavaScript XML) di React bekerja, yaitu menggabungkan Markup (HTML) dan JavaScript secara langsung.

Contoh Penggunaan Template Literals:

JavaScript

const appName = "TaskMaster";
const userName = USER_DATA.firstName; // 'Budi'
const remainingTasks = 5;

// Membuat UI Header & Data Statis
const headerHTML = `
    <header class="app-header">
        <h1>${appName}</h1>
        <p class="user-info">
            Selamat datang, <strong>${userName}</strong>!
            Anda memiliki ${remainingTasks} tugas tersisa.
        </p>
    </header>
`;

// String headerHTML kini mengandung markup dan data yang sudah terisi.

C. Relevansi ke React (Dasar JSX & Props)

  • JSX (JavaScript XML): Template Literals menunjukkan bagaimana kita bisa menulis markup (HTML) dan menyisipkan variabel JavaScript secara langsung. JSX adalah versi yang lebih elegan dan terstruktur dari ide ini.
  • Props (Properties): Objek USER_DATA yang dideklarasikan dengan const mencontohkan data statis yang akan dioper ke komponen lain. Di React, data statis yang dioper ke komponen sering disebut sebagai Props.

<body class="bg-gray-50">

    <div id="app" class="max-w-xl mx-auto mt-10 shadow-lg rounded-lg bg-white">

    <script>
        // Fungsi utama untuk menjalankan live coding
        function initializeApp() {
            console.log("Aplikasi TaskMaster Dimulai.");

            // --- LANGKAH 1: Deklarasi Variables (const) untuk Data Statis ---
            
            // Variabel konstanta (Primitive)
            const APP_TITLE = "TaskMaster Pro";
            const WELCOME_MESSAGE = "Selamat datang kembali!";

            // USER_DATA = { firstName: 'Ani' }; 
            
            // DEMO: Coba ubah properti objek (Diperbolehkan)
            USER_DATA.totalTasks = 15; 
            
            console.log("Data USER_DATA setelah diupdate properti:", USER_DATA); 

            // --- LANGKAH 2: Menggunakan Template Literals untuk Membuat Struktur UI ---
            
            // Membuat blok HTML dengan menyisipkan semua variabel
            const headerHTML = `
                <div class="p-6 border-b border-gray-200">
                    <h1 class="text-3xl font-bold text-gray-900">${APP_TITLE}</h1>
                    <p class="text-sm text-gray-600">${WELCOME_MESSAGE} ${USER_DATA.firstName} ${USER_DATA.lastName}</p>
                </div>
                <div class="p-6">
                    <h2 class="text-xl font-semibold mb-4 text-gray-700">Ringkasan Tugas</h2>
                    <div class="grid grid-cols-2 gap-4">
                        <div class="p-4 bg-blue-50 rounded-lg">
                            <p class="text-sm text-blue-700">Total Tugas</p>
                            <p class="text-3xl font-extrabold text-blue-900">${USER_DATA.totalTasks}</p>
                        </div>
                        <div class="p-4 bg-green-50 rounded-lg">
                            <p class="text-sm text-green-700">Sisa Tugas</p>
                            <p class="text-3xl font-extrabold text-green-900">${REMAINING_TASKS}</p>
                        </div>
                    </div>
                    <div class="mt-4 p-3 ${USER_DATA.isPremium ? 'bg-yellow-100 text-yellow-800' : 'bg-gray-100 text-gray-600'} rounded-lg text-sm font-medium">
                        Status Akun: ${USER_DATA.isPremium ? 'Premium (Akses Penuh)' : 'Dasar (Upgrade Now!)'}
                    </div>
                </div>
            `;

            if (appContainer) {
                // Menyisipkan string HTML ke dalam elemen #app
                appContainer.innerHTML = headerHTML;
            } else {
                console.error("Elemen #app tidak ditemukan.");
            }
        }

        document.addEventListener('DOMContentLoaded', initializeApp);
