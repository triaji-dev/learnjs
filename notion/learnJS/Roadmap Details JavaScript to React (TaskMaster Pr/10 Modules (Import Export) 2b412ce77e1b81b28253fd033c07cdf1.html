10. Modules (Import/Export) (15 Menit)

A. Kenapa Kita Perlu Modules?

Dalam aplikasi JavaScript modern, kita tidak lagi menempatkan semua kode dalam satu file <script> yang besar ("spaghetti code"). Ketika kode tumbuh, kita menghadapi masalah:

  • Global Scope Pollution: Semua variabel dan fungsi menjadi global, berpotensi menimpa satu sama lain.
  • Maintainability: Sangat sulit menemukan dan memperbaiki kode dalam file ribuan baris.
  • Dependency Management: Tidak jelas bagian kode mana yang membutuhkan bagian kode lain.

ES Modules (ESM), diperkenalkan di ES6, memecahkan masalah ini dengan memungkinkan kita:

  1. Memecah kode menjadi file-file kecil dan terisolasi (modules).
  1. Secara eksplisit mengekspor (export) bagian kode yang ingin dibagikan.
  1. Secara eksplisit mengimpor (import) bagian kode yang dibutuhkan dari file lain.

Syntax Export

Ada dua jenis export utama:

Tipe ExportDeskripsiContoh
Named ExportMengekspor beberapa nilai/fungsi dari satu module. Harus diimpor dengan nama yang sama dan kurung kurawal {}.export const APP_NAME = "TaskMaster";
Default ExportMengekspor satu nilai (fungsi, objek, atau kelas) sebagai default dari module. Dapat diimpor dengan nama apa pun (tanpa kurung kurawal).export default function init() { ... }

Syntax Import

Tipe ImportDeskripsiContoh
Named ImportMengimpor item spesifik dengan nama aslinya.import { APP_NAME } from './data.js';
Default ImportMengimpor item default.import initializeApp from './main.js';
Mix ImportMengimpor default dan named sekaligus.import initializeApp, { APP_NAME } from './main.js';

B. Fitur TaskMaster: Refactor Struktur File (MVC)

Kita akan memecah kode TaskMaster yang sebelumnya menjadi tiga file utama berdasarkan konsep MVC Sederhana (Model-View-Controller/Logika):

  1. data.js (Model): Mengelola data (state tugas) dan operasi data (tambah, hapus, filter, reduce).
    • Export: Fungsi manipulasi data (addTask, deleteTask, getTasks).
  1. view.js (View / UI): Mengelola tampilan dan interaksi dengan DOM/HTML.
    • Export: Fungsi untuk me-render tampilan (renderTaskList, renderStats).
  1. main.js (Controller / Logika): Menghubungkan Model dan View. Menangani event listener dan mengontrol aliran aplikasi.
    • Import: Semua fungsi dari data.js dan view.js.

C. Relevansi React: Struktur Komponen React

Konsep Modules adalah dasar dari arsitektur React:

  • Setiap file komponen (.jsx atau .js) adalah sebuah module.
  • Komponen utama selalu di-export sebagai default export (misalnya, export default function App()).
  • Fungsi pembantu (utilities) atau konstanta di-export sebagai named export (misalnya, export const PI = 3.14).
  • Setiap kali Anda menulis import React from 'react'; atau import { useState } from 'react';, Anda sedang menggunakan konsep Modules.

Live Coding: Refactor TaskMaster dengan Modules (15 Menit)

Kita akan merefaktor kode TaskMaster sebelumnya menjadi tiga file: data.js, view.js, dan main.js.

Struktur File:

taskmaster/
├── index.html
├── data.js <-- MODEL (Data & Logika Data)
├── view.js <-- VIEW (DOM/UI Manipulation)
└── main.js <-- CONTROLLER (Logika Utama & Event Handling)

Langkah-langkah Live Coding:

  1. Buat tiga file JS baru.
  1. Pindahkan data dan fungsi manipulasi data ke data.js dan export mereka.
  1. Pindahkan logika manipulasi DOM (rendering) ke view.js dan export mereka.
  1. Di main.js, import semua yang diperlukan dari data.js dan view.js.
  1. Ubah tag <script> di index.html menjadi <script type="module" src="main.js"></script> agar browser memahami sintaks import/export.

Contoh Code:

1. data.js (Model)

JavaScript

// A. Data State
let tasks = [
    { id: 1, title: 'Refactor dengan Modules', isCompleted: false },
    { id: 2, title: 'Siapkan Presentasi', isCompleted: true },
    { id: 3, title: 'Lunch Break', isCompleted: false },
];

// B. Logika Data (Exported Functions)
export function getTasks() {
    return tasks;
}

export function deleteTask(id) {
    // Memastikan id adalah angka
    const taskId = parseInt(id); 
    
    // Gunakan .filter() untuk immutability
    tasks = tasks.filter(task => task.id !== taskId);
    
    return tasks; // Kembalikan array baru
}

// Named Export lainnya
export const APP_TITLE = "TaskMaster Modular";

2. view.js (View)

JavaScript

// Import fungsi yang mungkin dibutuhkan dari file lain (jika ada, di sini tidak ada)

const taskListContainer = document.getElementById('task-list');

// Export fungsi rendering
export function renderTaskList(tasks, deleteCallback) {
    taskListContainer.innerHTML = '';
    
    if (tasks.length === 0) {
        taskListContainer.innerHTML = `<li class="p-4 text-gray-500 italic">Tidak ada tugas.</li>`;
        return;
    }

    tasks.forEach(task => {
        const isChecked = task.isCompleted ? 'line-through text-gray-500' : 'text-gray-900';
        
        const taskItemHTML = `
            <li class="flex justify-between items-center p-3 bg-white border rounded-lg shadow-sm">
                <span class="text-lg font-medium ${isChecked}">${task.title}</span>
                <button data-id="${task.id}" class="delete-btn bg-red-500 text-white px-3 py-1 text-sm rounded-md hover:bg-red-600 transition duration-150">
                    Hapus
                </button>
            </li>
        `;
        taskListContainer.innerHTML += taskItemHTML;
    });
    
    // Pasang Event Listeners (menggunakan callback dari main.js)
    document.querySelectorAll('.delete-btn').forEach(button => {
        button.addEventListener('click', (event) => {
            const taskId = event.target.getAttribute('data-id');
            deleteCallback(taskId); // Memanggil callback dari Controller (main.js)
        });
    });
}

3. main.js (Controller)

JavaScript

// Import Named Exports dari data.js
import { getTasks, deleteTask, APP_TITLE } from './data.js';

// Import Named Export dari view.js
import { renderTaskList } from './view.js';

// --- FUNGSI CONTROLLER ---
function handleTaskDeletion(id) {
    // 1. Logika: Perbarui data di Model
    deleteTask(id); 
    
    // 2. Tampilan: Ambil data baru dari Model, dan render ulang di View
    renderTaskList(getTasks(), handleTaskDeletion);
}

// Fungsi Inisialisasi Aplikasi (Default Export)
function initializeApp() {
    document.title = APP_TITLE;
    console.log(`Aplikasi ${APP_TITLE} dimulai.`);
    
    // Tampilkan daftar tugas awal
    // View memerlukan array tasks dan fungsi callback untuk delete
    renderTaskList(getTasks(), handleTaskDeletion);
}

// Jalankan inisialisasi aplikasi
initializeApp();
