A. Apa itu Optional Chaining?

Optional Chaining (?.) adalah fitur JavaScript (ES2020) yang memungkinkan Anda membaca nilai properti yang terletak jauh di dalam rantai objek tanpa harus memvalidasi setiap referensi di dalam rantai tersebut.

Intinya, ia menyediakan cara yang aman untuk mengakses properti dari sebuah objek, terutama ketika Anda tidak yakin apakah properti atau objek itu sendiri ada.

Masalah Tanpa ?.

Secara tradisional, ketika Anda mencoba mengakses properti dari objek yang mungkin null atau undefined, JavaScript akan memunculkan error yang fatal, menghentikan eksekusi kode (dan menyebabkan UI crash).

Contoh Masalah:

JavaScript

const user = {
    id: 1,
    name: 'Budi'
    // Properti 'address' TIDAK ADA
};

// Kode ini akan CRASH jika 'user.address' adalah undefined atau null
const city = user.address.city; 
// ‚ùå Error: Cannot read properties of undefined (reading 'city')

Untuk menghindari crash ini, kita harus melakukan validasi panjang:

JavaScript

// Solusi Tradisional (Verbose/Panjang)
const city = user && user.address && user.address.city;

Solusi dengan ?.

Operator ?. akan mengevaluasi rantai properti. Jika ada titik di rantai yang ditemukan sebagai null atau undefined, maka operator akan segera menghentikan evaluasi dan mengembalikan undefined, alih-alih memunculkan error.

JavaScript

const user = { id: 1, name: 'Budi' }; // user.address = undefined

// Solusi Modern (Sederhana dan Aman)
const city = user.address?.city; 
// Hasil: undefined (TIDAK ERROR!)

const zipCode = user?.address?.details?.zip;
// Hasil: undefined (TIDAK ERROR!)

B. Implementasi di Fitur "TaskMaster" (Handling Data)

Dalam aplikasi TaskMaster, kita sering berurusan dengan data yang berasal dari API. Data API tidak selalu konsisten atau lengkap, terutama di lingkungan staging atau ketika ada bug di backend.

Skenario: Kita memuat daftar tugas. Setiap tugas seharusnya memiliki objek metadata dan di dalamnya ada priority.

JavaScript

// 1. Tugas Sempurna
const task1 = { id: 1, title: 'Meeting', metadata: { priority: 'High', tags: ['work'] } };

// 2. Tugas yang Hilang Propertinya
const task2 = { id: 2, title: 'Review', metadata: null }; // metadata adalah null
const task3 = { id: 3, title: 'Draft', isCompleted: false }; // metadata TIDAK ADA

Akses Properti Tanpa Crash

Operasi AksesTanpa ?.Dengan ?.Hasil
task1.metadata.prioritytask1.metadata.prioritytask1.metadata?.priority'High'
task2.metadata.priority‚ùå CRASHtask2.metadata?.priorityundefined
task3.metadata.priority‚ùå CRASHtask3.metadata?.priorityundefined

Dengan ?., kita dapat memproses daftar tugas tanpa takut crash, dan kita dapat menggunakan default value yang aman jika properti tersebut hilang (lihat Nullish Coalescing di bagian live coding).

C. Relevansi React: Mencegah UI Crash

  • Data Asinkron: Komponen React sering di-render sebelum data dari API selesai dimuat. Jika Anda mencoba mengakses props.user.profile.firstName sebelum objek user ada (mungkin null atau undefined pada render pertama), UI akan Crash.
  • Optional Chaining adalah solusi paling bersih untuk mencegah UI Crash di React saat menangani props atau state yang mungkin asinkron atau tidak lengkap.

JavaScript

// Di dalam komponen React:
function UserProfile({ user }) {
    // Tanpa ?. akan Crash jika user = null
    // return <h1>Welcome, {user.name.firstName}!</h1> 

    // ‚úÖ Dengan Optional Chaining, akan menampilkan "Guest" jika user.name null/undefined
    const name = user?.name?.firstName || 'Guest'; 
    
    // Perhatikan: user?.tasks[0]?.title - Bekerja juga pada Array Index!
    const firstTask = user?.tasks?.[0]?.title; 

    return (
        <div className="profile">
            <h1>Halo, {name}</h1>
            <p>Tugas Pertama: {firstTask || 'Belum ada tugas'}</p>
        </div>
    );
}

Live Coding: Mencegah Error dengan Optional Chaining (15 Menit)

Kita akan membuat live coding sederhana untuk mendemonstrasikan bagaimana ?. melindungi kode kita saat memproses array tugas dengan data yang tidak konsisten.

Exercise untuk peserta:

  1. Buat array tugas dengan beberapa objek data yang tidak lengkap (sengaja hilangkan beberapa properti metadata).
  1. Tulis fungsi rendering yang mencoba mengakses properti yang dalam (misalnya, task.metadata.priority) tanpa ?.. Amati crash.
  1. Ubah kode untuk menggunakan ?. dan Nullish Coalescing Operator (??) untuk memberikan nilai default yang aman.

Contoh Code Starter (HTML/JS):

HTML

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Praktek: Optional Chaining (?.)</title>
    <script src="https://cdn.tailwindcss.com"></script>
</head>
<body class="bg-gray-100 p-8">

    <div id="app" class="max-w-xl mx-auto bg-white p-6 rounded-lg shadow-xl">
        <h1 class="text-3xl font-bold mb-6 text-gray-800">Demo Optional Chaining</h1>
        
        <div id="error-box" class="p-3 mb-4 bg-red-100 text-red-700 font-medium rounded hidden">
            Terjadi Error! Lihat console.
        </div>
        
        <ul id="task-list" class="space-y-3">
            </ul>
    </div>

    <script>
        // --- DATA DENGAN INKONSISTENSI ---
        const INCONSISTENT_TASKS = [
            // Tugas 1: Sempurna
            { id: 1, title: 'Setup Server', metadata: { priority: 'High', tag: 'DevOps' } },
            
            // Tugas 2: metadata adalah NULL
            { id: 2, title: 'Design UI', metadata: null },
            
            // Tugas 3: metadata TIDAK ADA (undefined)
            { id: 3, title: 'Write Documentation', isCompleted: false },
            
            // Tugas 4: metadata ada, tapi properti 'tag' tidak ada
            { id: 4, title: 'Debug Bug X', metadata: { priority: 'Medium' } }
        ];

        const taskListContainer = document.getElementById('task-list');
        const errorBox = document.getElementById('error-box');

        function renderTasks(tasks) {
            taskListContainer.innerHTML = '';
            errorBox.classList.add('hidden');
            
            tasks.forEach(task => {
                let tag = '';
                let priority = '';

                // --- BAGIAN KRITIS: AMAN DENGAN OPTIONAL CHAINING ---
                try {
                    // 1. Akses Properti Tanpa Optional Chaining (untuk demonstrasi crash)
                    // ‚ùå const crashTag = task.metadata.tag; 
                    // ‚ùå const crashPrio = task.metadata.priority; 
                    
                    // 2. ‚úÖ Akses Properti Dengan Optional Chaining
                    // Jika task.metadata null/undefined, eksekusi berhenti, hasilnya undefined
                    tag = task.metadata?.tag;
                    priority = task.metadata?.priority;

                    // 3. Gabungkan dengan Nullish Coalescing Operator (??)
                    // ?? memberikan nilai default HANYA jika hasilnya null atau undefined
                    tag = task.metadata?.tag ?? 'Tidak Ada Tag';
                    priority = task.metadata?.priority ?? 'Prioritas Biasa';
                    
                } catch (e) {
                    console.error("CRASH TERDETEKSI (Tanpa Optional Chaining):", e.message);
                    errorBox.classList.remove('hidden');
                    return; // Hentikan render jika crash
                }
                // --- AKHIR BAGIAN KRITIS ---

                const taskItemHTML = `
                    <li class="p-3 bg-gray-50 rounded-lg border flex justify-between">
                        <span class="font-medium">${task.title}</span>
                        <div class="space-x-2 text-sm">
                            <span class="bg-blue-100 text-blue-800 px-2 py-0.5 rounded">${priority}</span>
                            <span class="bg-gray-100 text-gray-600 px-2 py-0.5 rounded">${tag}</span>
                        </div>
                    </li>
                `;
                taskListContainer.innerHTML += taskItemHTML;
            });
        }

        // --- INISIALISASI ---
        document.addEventListener('DOMContentLoaded', () => {
             // Coba jalankan fungsi render. 
             // Untuk melihat crash, uncomment baris crashTag/crashPrio di atas.
             renderTasks(INCONSISTENT_TASKS);
        });
    </script>
</body>
</html>
