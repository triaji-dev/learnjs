13. Computed Property Names (15 Menit)

A. Konsep Dasar Computed Property Names

Computed Property Names adalah fitur ES6 yang memungkinkan Anda menggunakan ekspresi (seperti variabel atau hasil perhitungan) di dalam kurung siku [] saat mendefinisikan properti objek literal.

Ini memungkinkan nama key (kunci) dari sebuah properti objek ditentukan secara dinamis saat objek dibuat atau diperbarui.

Masalah Tanpa Computed Property Names

Secara tradisional, jika Anda ingin membuat properti objek menggunakan nama yang disimpan dalam sebuah variabel, Anda harus membuat objek terlebih dahulu, kemudian mengakses properti tersebut menggunakan notasi kurung siku:

JavaScript

const propertyName = 'priorityLevel';
const propertyValue = 'High';

// Solusi Tradisional (2 Langkah)
const task = {};
task[propertyName] = propertyValue;

/*
Hasil:
{ priorityLevel: 'High' }
*/

Solusi dengan Computed Property Names

Dengan [], kita dapat melakukan operasi yang sama hanya dalam satu deklarasi objek:

JavaScript

const propertyName = 'priorityLevel';
const propertyValue = 'High';

// Solusi Modern (1 Langkah)
const task = {
    title: 'Review Code',
    [propertyName]: propertyValue // <-- Menggunakan kurung siku untuk variabel
};

/*
Hasil:
{ title: 'Review Code', priorityLevel: 'High' }
*/

B. Implementasi di Fitur "TaskMaster" (Single Handler Form)

Kegunaan terbesar Computed Property Names adalah di Form Handling, terutama ketika kita ingin menggunakan satu fungsi (handleInput) untuk menangani perubahan pada banyak kolom input (seperti judul, tanggal, dan prioritas) sekaligus.

Skenario Aplikasi Form Tugas

Bayangkan Anda memiliki objek newTask yang menyimpan semua data formulir:

JavaScript

let newTask = {
    title: '',
    dueDate: '',
    priority: 'Low'
};

Setiap input di HTML diberi atribut name yang sesuai dengan key di objek newTask (misalnya, <input name="title">).

Ketika terjadi event change pada input, kita mendapatkan dua informasi penting:

  1. Nama field: Diambil dari event.target.name ('title', 'dueDate', atau 'priority').
  1. Nilai field: Diambil dari event.target.value.

Kita dapat menggunakan event.target.name sebagai nama properti yang dihitung!

JavaScript

// Satu fungsi untuk semua input
function handleInput(event) {
    const name = event.target.name;  // Contoh: 'title'
    const value = event.target.value; // Contoh: 'Pindah ke Modules'

    // Pembaruan Objek State secara Universal:
    newTask = {
        ...newTask, // Ambil semua properti lama (Immutability)
        
        // [name] adalah Computed Property Name
        // Jika name = 'title', maka { title: value }
        [name]: value 
    };

    console.log(newTask); 
}

Setiap kali input mana pun berubah, handleInput akan dipanggil, dan hanya properti yang sesuai (title, dueDate, atau priority) yang diperbarui, tanpa perlu membuat fungsi handler terpisah untuk setiap input.

C. Relevansi React: Multiple Input State di Form

Konsep ini adalah praktik standar untuk mengelola formulir di React, terutama ketika state formulir disimpan sebagai satu objek tunggal (useState).

JavaScript

import React, { useState } from 'react';

function TaskForm() {
    // State adalah objek tunggal
    const [taskData, setTaskData] = useState({
        title: '',
        priority: 'Low',
        dueDate: ''
    });

    // Single Handler Function
    const handleChange = (e) => {
        const { name, value } = e.target; // Destructuring event.target

        setTaskData(prevData => ({
            ...prevData, // Salin semua data state sebelumnya
            [name]: value // <-- Computed Property Name di sini!
        }));
    };

    return (
        <form>
            {/* Input 1: name="title" */}
            <input name="title" value={taskData.title} onChange={handleChange} /> 
            
            {/* Input 2: name="priority" */}
            <select name="priority" value={taskData.priority} onChange={handleChange}>
                {/* options... */}
            </select>
            
            {/* ...dan seterusnya untuk semua input... */}
        </form>
    );
}

        // Tampilkan state awal
        updateDisplay();
    </script>
