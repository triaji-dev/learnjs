14. LocalStorage & JSON (15 Menit)

A. Konsep Dasar Web Storage: localStorage

localStorage adalah mekanisme penyimpanan data sisi klien yang disediakan oleh browser. Ia memungkinkan aplikasi web menyimpan data secara permanen di dalam browser pengguna.

Karakteristik Utama localStorage

FiturDeskripsi
KapasitasLebih besar dari cookies, biasanya antara 5MB hingga 10MB per domain.
PersistensiData tidak memiliki tanggal kedaluwarsa dan akan tetap ada di browser bahkan setelah browser ditutup dan dibuka kembali.
Tipe DatalocalStorage hanya dapat menyimpan string. Jika Anda menyimpan objek atau array, Anda harus mengubahnya menjadi string terlebih dahulu (menggunakan JSON).
AksesData hanya dapat diakses oleh kode JavaScript pada domain yang sama.

Metode Utama localStorage

localStorage adalah objek global yang memiliki empat metode dasar:

  1. localStorage.**setItem(key, value)**: Menyimpan data (key dan value harus berupa string).
  1. localStorage.**getItem(key)**: Mengambil data yang tersimpan berdasarkan key. Mengembalikan string atau null jika key tidak ada.
  1. localStorage.**removeItem(key)**: Menghapus data berdasarkan key.
  1. localStorage.**clear()**: Menghapus semua data dari domain saat ini.

B. Peran JSON: Serialization dan Deserialization

Karena localStorage hanya dapat menyimpan string, kita membutuhkan JSON (JavaScript Object Notation) untuk mengkonversi data kompleks (seperti array tugas) menjadi string yang dapat disimpan, dan mengembalikannya lagi.

JSON menyediakan dua metode utama:

1. JSON.stringify() (Serialization)

Mengubah objek JavaScript atau array menjadi representasi string JSON yang dapat disimpan.

JavaScript

const tasks = [{ id: 1, title: 'Beli Kopi' }];
const tasksString = JSON.stringify(tasks); 
// Hasil: '[{"id":1,"title":"Beli Kopi"}]' (sebuah string)

localStorage.setItem('myTasks', tasksString);

2. JSON.parse() (Deserialization)

Mengubah string JSON kembali menjadi objek JavaScript atau array yang dapat digunakan.

JavaScript

const loadedString = localStorage.getItem('myTasks'); // Ambil string dari storage
const loadedTasks = JSON.parse(loadedString); 
// Hasil: [{ id: 1, title: 'Beli Kopi' }] (sebuah Array/Object JS)

C. Relevansi React: Persisting State / Custom Hooks

Dalam ekosistem React, konsep persisting state (menyimpan state agar tetap ada setelah refresh) sangat penting.

  • Persisting State: Ini adalah cara untuk membuat state aplikasi (misalnya, tasks) tetap ada di browser menggunakan localStorage.
  • Custom Hooks (useLocalStorage): Dalam aplikasi skala besar, developer sering membungkus logika JSON.stringify, JSON.parse, localStorage.setItem, dan localStorage.getItem ke dalam Custom Hook (misalnya useLocalStorage) untuk menjaga komponen tetap bersih dan logikanya dapat digunakan kembali.

Ketika menggunakan React, Anda biasanya akan:

  1. Load: Di awal (useEffect dengan dependensi kosong), cek localStorage. Jika ada, JSON.parse dan gunakan sebagai initial state.
  1. Save: Setiap kali state berubah (useEffect dengan [tasks] di dependency array), JSON.stringify dan localStorage.setItem data baru.

Live Coding: Fitur "Auto-Save" TaskMaster (15 Menit)

Kita akan membuat fitur Auto-Save sederhana di TaskMaster. Setiap kali tugas baru ditambahkan, data akan disimpan ke localStorage, dan ketika halaman dimuat ulang, data akan dimuat kembali.

Exercise untuk peserta:

  1. Buat fungsi saveTasks(tasks) yang menggunakan JSON.stringify dan localStorage.setItem.
  1. Buat fungsi loadTasks() yang menggunakan localStorage.getItem dan JSON.parse.
  1. Ubah state tugas awal (variabel tasks) untuk memanggil loadTasks() saat inisialisasi.
  1. Tambahkan logika untuk memanggil saveTasks() setiap kali ada penambahan tugas baru.

Contoh Code Starter (HTML/JS):

HTML

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Praktek: LocalStorage & JSON</title>

</head>
<body class="bg-gray-100 p-8">

    <div id="app" class="max-w-xl mx-auto bg-white p-6 rounded-lg shadow-xl">
        <h1 class="text-3xl font-bold mb-6 text-gray-800">TaskMaster: Auto-Save Demo</h1>
        
        <div class="mb-6 pb-4 border-b">
            <input type="text" id="new-task-title" placeholder="Tambahkan tugas baru..." 
                   class="w-3/4 p-2 border border-gray-300 rounded-l-md focus:outline-none focus:ring-2 focus:ring-indigo-500">
            <button id="add-task-btn" class="w-1/4 bg-indigo-600 text-white p-2 rounded-r-md hover:bg-indigo-700">
                Tambah
            </button>
        </div>

        <h2 class="text-xl font-semibold mb-3 text-gray-800">Daftar Tugas</h2>
        <ul id="task-list" class="space-y-3">
            </ul>
        
        <p class="mt-4 text-sm text-gray-500">Data tersimpan di LocalStorage.</p>
    </div>

    <script>
        const STORAGE_KEY = 'taskmaster_tasks';
        const taskListContainer = document.getElementById('task-list');
        const newTaskTitleInput = document.getElementById('new-task-title');
        const addTaskButton = document.getElementById('add-task-btn');

        let tasks = []; // State utama

        // --- 1. LOGIKA SIMPAN KE LOCALSTORAGE ---
        function saveTasks(currentTasks) {
            // ✅ Serialization: Array/Object JS -> String JSON
            const tasksJson = JSON.stringify(currentTasks);
            localStorage.setItem(STORAGE_KEY, tasksJson);
            console.log("Data berhasil disimpan ke LocalStorage.");
        }

        // --- 2. LOGIKA MUAT DARI LOCALSTORAGE ---
        function loadTasks() {
            // Ambil string dari LocalStorage
            const tasksJson = localStorage.getItem(STORAGE_KEY);
            
            if (tasksJson) {
                // ✅ Deserialization: String JSON -> Array/Object JS
                console.log("Data berhasil dimuat dari LocalStorage.");
                return JSON.parse(tasksJson);
            }
            
            // Jika tidak ada data, kembalikan array kosong sebagai default
            return [];
        }

        // --- FUNGSI UTAMA: RENDER DAN MANIPULASI DOM ---
        function renderTasks() {
            taskListContainer.innerHTML = '';
            
            if (tasks.length === 0) {
                taskListContainer.innerHTML = `<li class="p-4 text-center text-gray-500 italic">Daftar tugas kosong.</li>`;
                return;
            }

            tasks.forEach(task => {
                const li = document.createElement('li');
                li.className = 'p-3 bg-gray-50 rounded-lg border';
                li.textContent = task.title;
                taskListContainer.appendChild(li);
            });
        }

        function addTask() {
            const title = newTaskTitleInput.value.trim();
            if (title === '') return;

            const newId = tasks.length > 0 ? tasks[tasks.length - 1].id + 1 : 1;
            
            const newTask = {
                id: newId,
                title: title,
                isCompleted: false
            };

            // Perbarui state (Immutability)
            tasks = [...tasks, newTask];
            
            // 3. Simpan state baru ke LocalStorage
            saveTasks(tasks);
            
            // Render ulang dan bersihkan input
            renderTasks();
            newTaskTitleInput.value = '';
        }

        // --- INISIALISASI ---
        function initializeApp() {
            // 4. Muat data dari LocalStorage saat inisialisasi
            tasks = loadTasks(); 
            renderTasks();
            
            addTaskButton.addEventListener('click', addTask);
        }

        document.addEventListener('DOMContentLoaded', initializeApp);

    </script>
</body>
</html>
