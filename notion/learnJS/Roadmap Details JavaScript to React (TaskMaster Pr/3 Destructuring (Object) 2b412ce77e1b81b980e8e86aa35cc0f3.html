3. Destructuring (Object) (15 Menit)

A. Konsep Dasar Destructuring Objek di JavaScript

Destructuring adalah fitur ES6 (ECMAScript 2015) yang memungkinkan kita untuk "membongkar" properti dari sebuah objek (atau elemen dari array) dan menetapkannya ke variabel individual dengan sintaks yang ringkas.

Tujuan Utama:

  1. Mengambil Properti dengan Cepat: Mengurangi penulisan berulang (data.title, data.priority, dll.).
  1. Kode Lebih Bersih (Clean Code): Terutama saat menerima objek sebagai parameter fungsi.

Sintaks Dasar Destructuring Objek

Kita menggunakan kurung kurawal $(\{\})$ di sisi kiri penugasan (=) untuk mencocokkan nama properti objek yang ingin diambil.

Contoh 1: Mengambil Properti Objek Sederhana

Misalnya, kita memiliki objek tugas:

JavaScript

const task = {
    id: 't-001',
    title: 'Buat Materi Destructuring',
    priority: 'High',
    isComplete: false,
    dueDate: '2025-12-01'
};

// Cara Tradisional (sebelum Destructuring)
// const taskTitle = task.title;
// const taskPriority = task.priority;

// ‚úÖ Cara dengan Destructuring (ES6)
const { title, priority, isComplete } = task;

console.log(title);      // Output: Buat Materi Destructuring
console.log(priority);   // Output: High
console.log(isComplete); // Output: false

B. Aplikasi Destructuring untuk Fungsi

Ini adalah skenario penggunaan terpenting di JavaScript modern, khususnya di React. Destructuring membuat penanganan data dari parameter fungsi menjadi sangat efisien dan mudah dibaca.

Poin Relevansi React (Menerima Props di Komponen):

Dalam React, props (properties) selalu dikirim sebagai sebuah objek ke komponen. Dengan destructuring di parameter fungsi, kita bisa langsung mengambil properti yang kita butuhkan.

Contoh 2: Destructuring dalam Parameter Fungsi

Kita akan membuat fungsi untuk menampilkan detail tugas:

JavaScript

// Data yang akan dikirim (Objek Props)
const taskData = {
    title: 'Review PR Tim',
    priority: 'Urgent',
    assignee: 'Alice'
};

// ‚ùå Cara Tradisional (Mengakses properti satu per satu)
function displayTaskTraditional(props) {
    // const title = props.title;
    // const priority = props.priority;
    // ...
    console.log(`[Tradisional] Tugas: ${props.title}, Prioritas: ${props.priority}`);
}

// ‚úÖ Cara Modern (Destructuring Langsung di Parameter)
function displayTaskModern({ title, priority, assignee }) {
    // Variabel 'title', 'priority', dan 'assignee' sudah tersedia!
    console.log(`[Modern] Tugas: ${title}`);
    console.log(`[Modern] Prioritas: ${priority}`);
    console.log(`[Modern] Ditugaskan: ${assignee}`);
}

displayTaskTraditional(taskData);
// Output: [Tradisional] Tugas: Review PR Tim, Prioritas: Urgent

displayTaskModern(taskData);
/*
Output:
[Modern] Tugas: Review PR Tim
[Modern] Prioritas: Urgent
[Modern] Ditugaskan: Alice
*/

C. Fitur Lanjutan: Mengubah Nama Variabel & Nilai Default

1. Mengubah Nama Variabel (Aliasing)

Jika nama properti objek terlalu panjang atau sudah ada variabel dengan nama yang sama, kita bisa mengganti namanya menggunakan sintaks propertyName: newVariableName.

JavaScript

const user = {
    fullName: 'Jane Doe',
    userRole: 'Admin'
};

// Mengambil userRole tapi menyimpannya di variabel 'role'
const { fullName, userRole: role } = user;

console.log(fullName); // Output: Jane Doe
console.log(role);     // Output: Admin
// console.log(userRole); // -> Akan Error! userRole bukan variabel

2. Nilai Default

Kita bisa memberikan nilai default jika properti yang dicari tidak ada (atau bernilai undefined) di objek sumber, menggunakan sintaks propertyName = defaultValue.

JavaScript

const setting = { theme: 'dark' };

// Destructuring 'theme' dan 'fontSize'. 'fontSize' akan menggunakan default.
const { theme, fontSize = 16 } = setting;

console.log(theme);     // Output: dark
console.log(fontSize);  // Output: 16 (mengambil nilai default)

Live Coding (15 Menit)

Exercise untuk Peserta

Ajak peserta untuk membuat kode yang merefleksikan fitur "TaskMaster" seperti di screenshot, yaitu Ekstraksi Data Tugas dan relevansinya untuk menerima Props di Komponen (simulasi fungsi).

  1. Siapkan Objek Data Tugas: Buat objek bernama taskData yang berisi properti seperti id, title, priority, dan status.
  1. Lakukan Destructuring: Gunakan destructuring untuk mengambil properti title dan priority ke dalam variabel lokal.
  1. Simulasi Props (Parameter Fungsi): Buat fungsi TaskComponent yang menerima satu objek sebagai parameter, dan ubah agar menggunakan destructuring langsung di parameter untuk mengambil title, priority, dan properti baru assignee.
  1. Terapkan Nilai Default: Ubah destructuring di fungsi TaskComponent untuk memberikan nilai default Unassigned untuk properti assignee jika tidak disediakan.

HTML

<!DOCTYPE html>
<html lang="id">

        const taskData = {

        // -----------------------------------------------------------------

            <p><strong>Status:</strong> ${status}</p>
        `;

        // üöÄ LANGKAH 3 & 4: Destructuring di Parameter Fungsi (Simulasi Props)

        function TaskComponent({

        }) {

                    <p><strong>Prioritas:</strong> ${priority}</p>
                    <p><strong>Ditugaskan Kepada:</strong> ${assignee}</p>
                </div>
            `;
            
            // Catatan: Properti 'status' tidak diambil/digunakan di komponen ini, 
            // menunjukkan kita hanya mengambil yang dibutuhkan.
        }

        // Contoh 1: Mengirim Objek 'taskData' (tanpa 'assignee')
        console.log("--- Memanggil Komponen 1 (tanpa assignee) ---");
        TaskComponent(taskData); 
        // 'assignee' akan mengambil nilai default 'Unassigned'
        
        // Contoh 2: Mengirim Objek dengan semua properti
        const taskWithAssignee = {
            title: 'Deploy Hotfix',
            priority: 'Urgent',
            assignee: 'Budi Santoso'
        };
        
        console.log("--- Memanggil Komponen 2 (dengan assignee) ---");
        TaskComponent(taskWithAssignee);
        // 'assignee' akan mengambil nilai 'Budi Santoso'
        
    </script>
