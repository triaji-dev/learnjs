4. Ternary & Logical AND (&&) (15 Menit)

1. Penjelasan Materi dan Contoh (15 Menit)

Konsep ini adalah inti dari Conditional Rendering di React (dan framework UI modern lainnya), di mana kita perlu menampilkan atau mengubah elemen UI berdasarkan kondisi data (state atau props).

A. Ternary Operator (Operator Kondisional Tiga Bagian)

Ternary Operator adalah cara singkat untuk menulis pernyataan if-else sederhana. Ini sangat berguna ketika Anda perlu memilih satu dari dua nilai atau ekspresi berdasarkan sebuah kondisi.

Sintaks:

JavaScript

kondisi ? ekspresi_jika_TRUE : ekspresi_jika_FALSE;
BagianDeskripsi
kondisiEkspresi yang dievaluasi menjadi true atau false.
?Pemisah antara kondisi dan nilai jika true.
ekspresi_jika_TRUENilai atau kode yang dieksekusi jika kondisi true.
:Pemisah antara nilai jika true dan nilai jika false.
ekspresi_jika_FALSENilai atau kode yang dieksekusi jika kondisi false.

Contoh 1: Menentukan Teks atau Nilai

Kita ingin menampilkan label warna berdasarkan prioritas tugas.

JavaScript

const taskPriority = 'High';

// Menggunakan if-else tradisional:
// let priorityColor;
// if (taskPriority === 'High') {
//     priorityColor = 'red';
// } else {
//     priorityColor = 'blue';
// }

// âœ… Menggunakan Ternary Operator:
const priorityColor = taskPriority === 'High' ? 'red' : 'blue';

console.log(priorityColor); // Output: red

Relevansi React:

Ternary sering digunakan untuk menetapkan nilai atribut (seperti className atau style) atau untuk memilih komponen mana yang akan ditampilkan.

JavaScript

// Contoh dalam konteks JSX (di React):
<div className={taskStatus === 'Done' ? 'bg-green-100' : 'bg-yellow-100'}>
    {taskStatus}
</div>

B. Logical AND Operator (&&)

Operator Logical AND (&&) mengembalikan nilai ekspresi pertama yang bernilai falsy (false, 0, "", null, undefined) atau, jika semua ekspresi bernilai truthy, ia mengembalikan nilai ekspresi terakhir.

Di JavaScript modern, ini umum digunakan sebagai bentuk singkatan untuk pernyataan if sederhana, terutama dalam Conditional Rendering (menampilkan elemen HANYA jika kondisinya benar).

Sintaks:

JavaScript

kondisi_truthy && ekspresi_yang_dieksekusi_jika_TRUE

Bagaimana cara kerjanya (Short-Circuit Evaluation):

  1. Jika kondisi_truthy bernilai true, JavaScript akan melihat ekspresi berikutnya (ekspresi_yang_dieksekusi_jika_TRUE) dan mengembalikan nilainya.
  1. Jika kondisi_truthy bernilai false, JavaScript akan langsung berhenti (short-circuit) dan mengembalikan nilai false tanpa melihat ekspresi kedua.

Contoh 2: Conditional Rendering

Dalam aplikasi TaskMaster, kita ingin menampilkan tombol "Edit" hanya jika status tugas bukan Done.

JavaScript

const taskStatus1 = 'In Progress';
const taskStatus2 = 'Done';

// Kondisi 1: taskStatus1 BUKAN 'Done' (True)
const showEditButton1 = taskStatus1 !== 'Done' && 'Tampilkan Tombol Edit';

// Kondisi 2: taskStatus2 BUKAN 'Done' (False)
const showEditButton2 = taskStatus2 !== 'Done' && 'Tampilkan Tombol Edit';

console.log(showEditButton1); // Output: Tampilkan Tombol Edit (nilai dari ekspresi kedua)
console.log(showEditButton2); // Output: false (nilai dari ekspresi pertama, karena false)

Relevansi React:

Dalam JSX (React), jika ekspresi pertama (kondisi_truthy) bernilai true, ekspresi kedua yang berupa markup HTML/JSX akan dieksekusi dan ditampilkan. Jika ekspresi pertama false, React akan mengabaikan ekspresi kedua (markup) dan tidak menampilkan apapun, sehingga terjadi penyembunyian elemen.

JavaScript

// Contoh dalam konteks JSX (di React):
// { taskStatus !== 'Done' && <button onClick={handleEdit}>Edit</button> }

Live Coding (15 Menit)

Exercise untuk Peserta

Ajak peserta untuk mengimplementasikan Conditional Rendering menggunakan Ternary dan && pada data tugas (Fitur TaskMaster: Label Prioritas & Status).

  1. Siapkan Data Tugas: Deklarasikan objek task dengan properti priority (High, Medium, atau Low) dan status (To Do, In Progress, atau Done).
  1. Terapkan Ternary untuk Class Styling: Gunakan Ternary Operator untuk menetapkan class CSS (warna) berdasarkan nilai task.priority.
  1. Terapkan Ternary untuk Logika Kompleks (2 Tingkat): Gunakan Ternary yang bersarang (atau buat variabel bantu) untuk menangani tiga kondisi prioritas (High, Medium, Low).
  1. Terapkan Logical AND (&&) untuk Conditional Rendering: Gunakan && untuk menampilkan tombol Edit hanya jika status tugas bukan Done.

Contoh Code (Untuk panduan live coding)

HTML

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Praktek: Ternary & Logical AND</title>
    <script src="https://cdn.tailwindcss.com"></script>
    <style>
        .badge {
            display: inline-block;
            padding: 2px 8px;
            border-radius: 9999px;
            font-size: 0.75rem;
            font-weight: 600;
        }
        .badge-red { background-color: #fca5a5; color: #7f1d1d; } /* High */
        .badge-yellow { background-color: #fde68a; color: #92400e; } /* Medium */
        .badge-blue { background-color: #bfdbfe; color: #1e40af; } /* Low/Default */
        .badge-green { background-color: #a7f3d0; color: #065f46; } /* Done Status */
    </style>
</head>
<body>

    <div id="app" class="p-8 max-w-lg mx-auto">
        <h1 class="text-2xl font-bold mb-6">TaskMaster - Conditional Rendering</h1>
        </div>

    <script>
        // Data Tugas untuk Simulasi Conditional Rendering
        const taskData = {
            title: 'Selesaikan Desain UI Mobile',
            priority: 'High', // Coba ubah ke 'Medium' atau 'Low'
            status: 'In Progress', // Coba ubah ke 'Done' atau 'To Do'
            isUrgent: true
        };

        function renderTaskDetail(task) {
            
            // -----------------------------------------------------------------
            // ðŸš€ LANGKAH 1 & 2: Ternary Operator untuk 2 Kondisi (Priority Color)
            // -----------------------------------------------------------------
            
            // Menentukan warna badge (2 kondisi: High atau Default)
            const priorityBadgeClass = task.priority === 'High' 
                ? 'badge-red' 
                : 'badge-blue';

            // -----------------------------------------------------------------
            // ðŸš€ LANGKAH 3: Ternary Bersarang/Bantuan untuk 3 Kondisi (Lebih Kompleks)
            // -----------------------------------------------------------------

            // Menentukan warna badge untuk 3 kondisi (High, Medium, Low/Default)
            const complexPriorityClass = task.priority === 'High' 
                ? 'badge-red' 
                : (task.priority === 'Medium' 
                    ? 'badge-yellow' 
                    : 'badge-blue'); 
            
            // Status badge (contoh ternary sederhana lagi)
            const statusClass = task.status === 'Done' ? 'badge-green' : 'badge-yellow';


            // -----------------------------------------------------------------
            // ðŸš€ LANGKAH 4: Logical AND (&&) untuk Conditional Rendering Tombol
            // -----------------------------------------------------------------
            
            // Variabel ini akan berisi string (markup) atau false.
            const editButtonMarkup = task.status !== 'Done' && 
                `<button class="bg-indigo-500 hover:bg-indigo-600 text-white py-1 px-3 rounded-md text-sm ml-4">
                    Edit Tugas
                </button>`;

            // Jika task.status adalah 'Done', task.status !== 'Done' menjadi false.
            // Ekspresi keseluruhan menjadi false, dan tombol tidak dirender.
            // Jika task.status BUKAN 'Done', ekspresi menjadi string markup, dan dirender.

            
            // Gabungkan semua ke dalam Template Literal
            const taskHTML = `
                <div class="p-5 border border-gray-200 rounded-lg shadow-md mb-4">
                    <div class="flex items-center justify-between mb-3">
                        <h2 class="text-xl font-semibold">${task.title}</h2>
                        
                        ${task.isUrgent && '<span class="text-red-500 font-bold text-xs ml-2">URGENT!</span>'}
                    </div>

                    <div class="flex space-x-2 mb-4">
                        <span class="badge ${complexPriorityClass}">
                            Prioritas: ${task.priority}
                        </span>

                        <span class="badge ${statusClass}">
                            Status: ${task.status}
                        </span>
                    </div>

                    <div class="flex justify-end">
                        ${editButtonMarkup} 
                        <button class="bg-green-500 hover:bg-green-600 text-white py-1 px-3 rounded-md text-sm">
                            ${task.status === 'Done' ? 'Arsipkan' : 'Tandai Selesai'}
                        </button>
                    </div>

                </div>
            `;
            
            return taskHTML;
        }

        // Tampilkan hasil rendering ke halaman
        const appContainer = document.getElementById('app');
        appContainer.innerHTML += renderTaskDetail(taskData);

        // Tambahkan satu tugas yang Selesai untuk demonstrasi perbandingan
        const doneTask = {
            title: 'Review Materi Lama',
            priority: 'Low',
            status: 'Done', // <-- Status Done, Tombol Edit akan hilang
            isUrgent: false
        };
        appContainer.innerHTML += renderTaskDetail(doneTask);
        
    </script>
</body>
