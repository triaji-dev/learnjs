6. Immutability (Spread ...) (30 Menit)

1. Penjelasan Materi dan Contoh (15 Menit)

A. Konsep Dasar Immutability

Immutability (ketidak-mampuan-diubah) adalah konsep pemrograman fundamental, terutama dalam pengembangan aplikasi modern seperti React, yang berarti bahwa setelah sebuah data (objek atau array) dibuat, data tersebut tidak boleh diubah.

  • Mutating (Diubah): Langsung memodifikasi data asli (Array atau Object). Contoh: array.push(), object.property = newValue.
  • Immutable (Tidak Diubah): Membuat salinan baru dari data asli, lalu memodifikasi salinan tersebut. Data lama tetap utuh.

Mengapa Immutability Penting di React (State Management)?

Di React, jika Anda memodifikasi Array atau Object yang merupakan state secara langsung (mutating), React tidak akan mendeteksi perubahan tersebut karena referensi memorinya masih sama. React perlu melihat referensi baru untuk memicu proses re-render.

B. Spread Operator (...)

Spread Operator (...) adalah fitur ES6 yang merupakan cara paling umum dan paling bersih untuk mencapai Immutability (membuat salinan baru dari Array atau Object).

Tanda titik tiga (...) digunakan untuk mengembangkan (menyebar) elemen-elemen dari Iterable (seperti Array) atau properti dari Object ke dalam Array atau Object baru.

1. Immutability pada Array (Fitur "Tambah Tugas")

Untuk menambahkan elemen ke Array secara Immutable, kita tidak menggunakan push() (yang mutating). Kita membuat Array baru:

Detail Implementasi: Fungsi addTask tidak pakai .push(), tapi membuat array baru [...old, new].

JavaScript

const initialTasks = ['Makan', 'Tidur'];
const newTask = 'Coding';

// ‚ùå Cara Mutating (HINDARI di React State)
// initialTasks.push(newTask); 
// console.log(initialTasks); // [ 'Makan', 'Tidur', 'Coding' ] <-- Array asli berubah

// ‚úÖ Cara Immutable menggunakan Spread Operator
const newTaskList = [...initialTasks, newTask];

console.log(newTaskList);   // Output: [ 'Makan', 'Tidur', 'Coding' ]
console.log(initialTasks);  // Output: [ 'Makan', 'Tidur' ] <-- Array asli tetap utuh!

Keterangan: ...initialTasks menyalin semua elemen lama, kemudian newTask ditambahkan di akhir.

2. Immutability pada Objek (Mengubah Properti)

Untuk mengubah properti pada Object secara Immutable, kita juga menggunakan Spread Operator. Ini sangat penting saat memperbarui state yang merupakan Object.

JavaScript

const user = { 
    name: 'Budi', 
    age: 25, 
    isAdmin: false 
};

// ‚ùå Cara Mutating
// user.isAdmin = true; // Objek asli 'user' berubah

// ‚úÖ Cara Immutable menggunakan Spread Operator
const updatedUser = { 
    ...user,        // 1. Salin semua properti lama dari objek 'user'
    isAdmin: true,  // 2. Timpa/Update properti yang ingin diubah (isAdmin)
    email: 'budi@mail.com' // 3. Tambahkan properti baru
};

console.log(updatedUser); 
/* Output: 
{ name: 'Budi', age: 25, isAdmin: true, email: 'budi@mail.com' } 
*/
console.log(user); 
/* Output: 
{ name: 'Budi', age: 25, isAdmin: false } <-- Objek asli tetap utuh!
*/
