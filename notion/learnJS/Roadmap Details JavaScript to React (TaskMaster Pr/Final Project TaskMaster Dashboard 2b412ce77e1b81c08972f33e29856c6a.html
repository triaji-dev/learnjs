Instruksi Proyek Akhir: TaskMaster Dashboard

Judul Proyek: TaskMaster Dashboard

Tujuan: Membuat aplikasi manajemen tugas sederhana (To-Do List) yang memanfaatkan semua fitur JavaScript modern dan mengaplikasikan prinsip pengelolaan state dan rendering layaknya framework modern (misalnya, React), meskipun dibangun dari dasar.

üìù Fitur Utama

Aplikasi harus memungkinkan pengguna untuk:

  1. Menambah Tugas baru (Judul, Prioritas, Deadline).
  1. Menghapus Tugas.
  1. Mengubah Status Tugas menjadi "Selesai" atau "Belum".
  1. Memfilter daftar tugas berdasarkan Status (Done / To Do).
  1. Menampilkan Statistik tugas secara real-time.
  1. Menyimpan Data secara otomatis di browser.

üß± Struktur Proyek & Implementasi Topik

Berikut adalah panduan langkah demi langkah, beserta relevansi topiknya:

1. Persiapan Data & Struktur HTML (Topik 1, 10, 16)

  • Buat struktur folder dengan file: index.html, style.css, dan folder js/.
  • Di folder js/, buat file:
    • data.js: Berisi data array tugas awal (untuk simulasi load data).
    • model.js: Berisi semua fungsi logika data (addTask, deleteTask, toggleStatus, dll).
    • ui.js: Berisi fungsi untuk membuat dan merender elemen UI (createTaskCard).
    • main.js: File utama yang menginisialisasi aplikasi dan menangani event.
  • Implementasi Topik 1: Gunakan template literals (``) secara ekstensif di ui.js untuk membuat card tugas (HTML template).
  • Implementasi Topik 16: Pastikan deadline tugas diformat dengan baik ("Jumat, 3 Mei 2024") sebelum ditampilkan.

2. Rendering Tugas Awal (Topik 2, 3, 4, 13)

  • Implementasi Topik 2: Di ui.js, buat fungsi createTaskCard(task) sebagai Arrow Function yang menerima objek tugas dan mengembalikan string HTML (meniru Functional Component React).
  • Implementasi Topik 3: Gunakan Destructuring saat menerima objek tugas di dalam createTaskCard ({title, priority, isDone}).
  • Implementasi Topik 4: Gunakan Ternary Operator di dalam template literal untuk:
    • Menampilkan label Prioritas (Misal: warna badge berbeda untuk High).
    • Menampilkan tombol "Selesai" jika status isDone adalah false, atau "Batal Selesai" jika true.
  • Implementasi Topik 13: Saat membuat form input, gunakan Computed Property Names untuk menangani input ganda (Judul, Prioritas) dalam satu fungsi handler (misalnya: $handleInput(e)$ yang memperbarui objek state sementara).

3. Memproses Daftar Tugas (Topik 5, 6, 7, 8)

Ini adalah inti dari rendering dan state management.

  • Implementasi Topik 5: Di main.js, buat fungsi renderTasks(tasks). Gunakan Array Map (Array.map()) untuk mengubah array data tugas menjadi array string HTML/elemen UI, lalu masukkan ke dalam DOM.
  • Implementasi Topik 6 (Immutability): Semua operasi modifikasi tugas (Tambah, Hapus, Ubah Status) di model.js HARUS menjaga immutability.
    • Menambah Tugas: Gunakan Spread Operator (...) untuk membuat array tugas baru, bukan push().
    • Menghapus/Ubah: Gunakan map() atau filter() untuk membuat array baru, bukan memodifikasi elemen array yang lama. * Implementasi Topik 7: Hubungkan semua tombol di UI (Tambah, Hapus, Toggle Status) ke fungsi handler di main.js menggunakan event listeners. Pastikan fungsi toggle status dilewatkan sebagai Callback ke tombol yang dibuat.
  • Implementasi Topik 8: Tambahkan fungsi Filter Tugas. Gunakan Array Filter (Array.filter()) untuk hanya menampilkan tugas yang statusnya "Selesai" atau "Belum Selesai" berdasarkan tombol filter yang diklik pengguna.

4. Statistik dan Logika Kompleks (Topik 9, 12)

  • Implementasi Topik 9: Buat fungsi calculateStats(tasks) di model.js. Gunakan Array Reduce (Array.reduce()) untuk menghitung jumlah total tugas, tugas selesai, dan tugas belum selesai dalam satu kali iterasi. Tampilkan hasil ini di Dashboard.
  • Implementasi Topik 12: Saat mengambil data dummy dari data.js (atau API sungguhan jika Anda mengembangkannya), simulasikan data yang hilang/tidak lengkap. Gunakan Optional Chaining (?.) di ui.js saat mengakses properti tugas (misalnya: $task.user?.name$) untuk mencegah error jika properti tidak ada.

5. Persistence & Async/Await (Topik 11, 14, 15)

  • Implementasi Topik 11 (Async/Await): Di main.js, buat fungsi async untuk mengambil data tugas awal. Simulasikan keterlambatan jaringan menggunakan setTimeout di dalam Promise atau simulasi API call agar Anda bisa menggunakan await. Tampilkan status "Loading..." sebelum data muncul.
  • Implementasi Topik 14: Setelah setiap operasi modifikasi (Tambah, Hapus, Ubah Status), simpan array tugas terbaru ke LocalStorage menggunakan JSON.stringify(). Saat aplikasi di-load (setelah simulasi async), cek LocalStorage dan gunakan data dari sana jika ada. Gunakan JSON.parse() untuk mengambilnya.
  • Implementasi Topik 15 (Value vs Reference): Saat menambahkan tugas baru, pastikan Anda tidak menyalin objek input baru secara by reference. Buat deep copy dari payload form untuk menghindari masalah duplikasi atau modifikasi yang tidak disengaja.

// Topik 6: Immutability (Menggunakan Spread Operator)
export const addTask = (tasks, newTaskData) => {
    const newId = Date.now().toString();
    const newTask = {
        id: newId,
        isDone: false,
        ...newTaskData // Mengambil data title, priority, deadline dari form
    };
    // Mengembalikan array baru (immutability)
    return [...tasks, newTask];
};

// Topik 6: Immutability (Menggunakan Filter)
export const deleteTask = (tasks, taskId) => {
    // Mengembalikan array baru tanpa tugas dengan taskId yang cocok
    return tasks.filter(task => task.id !== taskId);
};

// Topik 6: Immutability (Menggunakan Map)
export const toggleTaskStatus = (tasks, taskId) => {
    // Mengembalikan array baru dengan status tugas yang diubah
    return tasks.map(task => {
        if (task.id === taskId) {
            // Mengembalikan objek tugas baru (immutability)
            return { ...task, isDone: !task.isDone };
        }
        return task;
    });
};

// --- FUNGSI PENGHITUNGAN STATISTIK ---

// Topik 9: Array Reduce
export const calculateStats = (tasks) => {
    // Menggunakan reduce untuk menghitung semua statistik dalam satu iterasi
    return tasks.reduce((acc, task) => {
        // Total tugas
        acc.total += 1;
        
        // Cek status selesai
        if (task.isDone) {
            acc.done += 1;
        } else {
            acc.todo += 1;
        }

        return acc;
    }, { total: 0, done: 0, todo: 0 }); // Nilai awal (accumulator)
};

const formatDate = (timestamp) => {
    try {
        const date = new Date(timestamp);
        const options = { weekday: 'long', day: 'numeric', month: 'short', year: 'numeric' };
        // Menggunakan "en-US" untuk menghindari masalah i18n yang kompleks, hasilnya konsisten
        return date.toLocaleDateString('id-ID', options);
    } catch (e) {
        return 'N/A';
    }
};

export const createTaskCard = ({ id, title, priority, isDone, deadline }) => {

Main Application Logic js/main.js
// Topik 10: Modules (Import)
import { initialTasks } from './data.js';
import { addTask, deleteTask, toggleTaskStatus, calculateStats } from './model.js';
import { createTaskCard, renderTasks, renderStats, renderLoading, clearForm } from './ui.js';

// Topik 14: LocalStorage key
const STORAGE_KEY = 'taskMaster_tasks';

// STATE UTAMA APLIKASI
let taskState = [];
let currentFilter = 'all'; // all, done, todo

// Cache DOM Elements
const taskListContainer = document.getElementById('task-list-container');
const taskForm = document.getElementById('task-form');
const filterButtons = document.querySelectorAll('.filter-btn');

// --- FUNGSI UTAMA RENDERING DAN UPDATE STATE ---

// Fungsi utama yang dipanggil setelah state berubah
const updateUI = () => {
    // 1. Filter tugas (Topik 8: Array Filter)
    let filteredTasks = taskState;
    if (currentFilter === 'done') {
        filteredTasks = taskState.filter(task => task.isDone);
    } else if (currentFilter === 'todo') {
        filteredTasks = taskState.filter(task => !task.isDone);
    }

    // 2. Render daftar tugas
    renderTasks(filteredTasks, taskListContainer);

    // 3. Hitung dan render statistik (Topik 9: Array Reduce)
    const stats = calculateStats(taskState);
    renderStats(stats);
    
    // 4. Simpan ke LocalStorage (Topik 14: LocalStorage & JSON)
    localStorage.setItem(STORAGE_KEY, JSON.stringify(taskState));
};

// --- FUNGSI ASYNC LOAD DATA ---

// Topik 11: Async / Await
const loadInitialData = async () => {
    renderLoading(true); // Tampilkan status loading

    // Cek LocalStorage (Topik 14: LocalStorage & JSON)
    const storedTasksJSON = localStorage.getItem(STORAGE_KEY);
    
    // Simulasi delay jaringan/API call
    await new Promise(resolve => setTimeout(resolve, 1500)); 

    if (storedTasksJSON) {
        // Data ditemukan di LocalStorage
        taskState = JSON.parse(storedTasksJSON);
    } else {
        // Data dari dummy file
        taskState = initialTasks;
    }

    renderLoading(false); // Sembunyikan status loading
    updateUI();
};

// --- HANDLER EVENTS ---

// Handler Submit Form (Topik 7, 13, 15)
taskForm.addEventListener('submit', (e) => {
    e.preventDefault();
    
    const formData = new FormData(taskForm);
    const newTaskData = {};
    
    // Topik 13: Computed Property Names (mengambil nama input secara dinamis)
    for (let [name, value] of formData.entries()) {
        newTaskData[name] = value;
    }
    
    // Topik 15: Value vs Reference. Pastikan kita hanya mengambil nilai (sudah dilakukan via FormData)
    // Jika kita menggunakan objek langsung dari input (misalnya state form di React), kita perlu deep copy.
    // Di sini, newTaskData sudah menjadi objek VALUE baru.
    
    // Panggil model untuk menambah tugas (Topik 6: Immutability)
    taskState = addTask(taskState, newTaskData);

    clearForm(taskForm);
    updateUI();
});

// Handler Klik Tombol pada List (Hapus, Toggle Status)
// Topik 7: Events & Callbacks
taskListContainer.addEventListener('click', (e) => {
    const target = e.target;
    const taskId = target.getAttribute('data-id');
    const action = target.getAttribute('data-action');

    if (!taskId || !action) return;

    if (action === 'delete') {
        // Konfirmasi Hapus (menggunakan modal simulasi, bukan alert/confirm)
        if (window.confirm('Anda yakin ingin menghapus tugas ini?')) {
            // Panggil model untuk menghapus (Topik 6: Immutability)
            taskState = deleteTask(taskState, taskId);
            updateUI();
        }
    } else if (action === 'toggle') {
        // Panggil model untuk mengubah status (Topik 6: Immutability)
        taskState = toggleTaskStatus(taskState, taskId);
        updateUI();
    }
});

// Handler Filter (Topik 8: Array Filter)
filterButtons.forEach(button => {
    button.addEventListener('click', () => {
        // Hapus styling aktif dari semua tombol
        filterButtons.forEach(btn => btn.classList.remove('bg-indigo-700', 'text-white'));
        
        // Atur filter baru
        currentFilter = button.getAttribute('data-filter');
        
        // Tambahkan styling aktif pada tombol yang diklik
        button.classList.add('bg-indigo-700', 'text-white');
        
        // Panggil updateUI untuk me-render ulang
        updateUI();
    });
});


// --- INISIALISASI APLIKASI ---
window.onload = loadInitialData;
